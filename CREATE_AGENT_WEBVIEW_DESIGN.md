# 创建 Agent Webview 方案设计

## 一、为什么需要消息通信？

### 问题背景
Webview 是一个独立的 HTML 页面，运行在隔离的上下文中（类似 iframe）。它：
- ❌ **不能直接访问 VSCode API**（如 vscode.window, vscode.workspace）
- ❌ **不能直接访问 Node.js API**（如 fs, path）
- ❌ **不能直接调用扩展代码**（如 AgentManager）

### 消息通信解决的问题

1. **数据传递**：表单数据从 Webview 传到扩展
2. **业务逻辑执行**：扩展调用 AgentManager.createAgent()
3. **结果反馈**：创建成功/失败的消息返回给 Webview
4. **实时验证**：Webview 请求扩展检查 ID 是否已存在
5. **状态同步**：创建成功后，扩展通知 Webview 关闭，并更新 Agent 列表

### 消息通信流程
```
用户填写表单 
  ↓
Webview 收集数据 
  ↓
postMessage 发送给扩展 
  ↓
扩展接收消息 
  ↓
调用 AgentManager.createAgent() 
  ↓
创建成功/失败 
  ↓
postMessage 发送结果给 Webview 
  ↓
Webview 显示结果（成功提示/错误信息）
```

## 二、整体架构

### 方案选择：独立 Webview

**不采用**：在现有聊天 Webview 中添加模态框
**采用**：创建独立的 CreateAgentPanel

**理由**：
1. ✅ 职责分离，聊天面板专注于聊天
2. ✅ 更好的 UI/UX，不受聊天面板布局限制
3. ✅ 独立开发和测试
4. ✅ 未来可扩展为 Agent 管理面板（创建、编辑、删除、查看）

### 文件结构
```
src/
  ├── createAgentPanel.ts          # 新增：CreateAgentPanel 类
  └── chatController.ts            # 修改：调用 CreateAgentPanel

webview/
  ├── create-agent.html            # 新增：创建 Agent 页面
  ├── create-agent.js              # 新增：表单逻辑和消息通信
  └── create-agent.css             # 新增：样式（可选，复用 styles.css）
```

### 表单字段

| 字段 | 类型 | 必填 | 验证规则 | 说明 |
|------|------|------|----------|------|
| Agent ID | 输入框 | ✓ | 小写字母、数字、连字符，不能重复 | 实时验证是否已存在 |
| 显示名称 | 输入框 | ✓ | 非空 | Agent 的显示名称 |
| Emoji | 输入框/选择器 | ✓ | 单个 Emoji | 可以手动输入或选择 |
| 角色 | 卡片选择 | ✓ | 6个预设角色 | 全栈、前端、后端、DevOps、测试、架构师 |
| 自定义描述 | 文本域 | ✗ | 可选 | 追加到角色模板后面 |

### 交互细节

1. **ID 实时验证**：
   - 输入时防抖（300ms）
   - 发送消息给扩展检查是否存在
   - 显示 ✓ 可用 / ✗ 已存在

2. **角色选择**：
   - 卡片式选择（6个预设角色）
   - 点击卡片高亮选中
   - 显示角色描述和特点

3. **实时预览**：
   - 右侧预览区域实时显示 Agent 卡片
   - 包括 Emoji、名称、ID、角色

4. **表单验证**：
   - 必填字段为空时，创建按钮禁用
   - ID 已存在时，创建按钮禁用
   - 显示错误提示

5. **创建流程**：
   - 点击"创建 Agent"按钮
   - 显示加载状态（按钮变为"创建中..."）
   - 创建成功：显示成功提示，关闭面板，刷新 Agent 列表
   - 创建失败：显示错误信息，保持面板打开

## 四、消息协议

### Webview → Extension

```typescript
// 1. 检查 ID 是否存在
{
  type: 'checkAgentId',
  id: string
}

// 2. 创建 Agent
{
  type: 'createAgent',
  config: {
    id: string,
    name: string,
    emoji: string,
    role: string,
    description?: string
  }
}

// 3. 取消创建
{
  type: 'cancelCreateAgent'
}

// 4. 请求初始化数据
{
  type: 'requestInitData'
}
```

### Extension → Webview

```typescript
// 1. ID 检查结果
{
  type: 'agentIdCheckResult',
  id: string,
  exists: boolean
}

// 2. 创建结果
{
  type: 'createAgentResult',
  success: boolean,
  error?: string,
  agentId?: string
}

// 3. 初始化数据
{
  type: 'initData',
  roles: Array<{
    id: string,
    name: string,
    description: string,
    icon: string
  }>,
  existingAgents: Array<{
    id: string,
    name: string,
    emoji: string
  }>
}
```

## 五、实现步骤

### Phase 1：基础 Webview 和表单（2-3小时）

**目标**：创建基础的 Webview 面板和表单

**任务**：
1. 创建 `CreateAgentPanel` 类（参考 ChatPanel）
2. 创建 `create-agent.html`（基础表单）
3. 创建 `create-agent.js`（表单逻辑和消息通信）
4. 创建 `create-agent.css`（样式，复用 styles.css）
5. 在 `chatController.ts` 中集成（替换现有的 InputBox 方式）

**验收标准**：
- ✓ 点击"创建 Agent"按钮，打开 Webview 面板
- ✓ 表单包含所有字段（ID、名称、Emoji、角色、描述）
- ✓ 点击"创建"按钮，能够发送消息给扩展
- ✓ 扩展能够接收消息并调用 AgentManager.createAgent()
- ✓ 创建成功后，关闭面板

### Phase 2：实时验证和预览（1-2小时）

**目标**：实现 ID 实时验证和预览区域

**任务**：
1. 实现 ID 实时验证（防抖 300ms）
2. 扩展端实现 `checkAgentId` 消息处理
3. 实现预览区域（右侧显示 Agent 卡片）
4. 表单验证（必填字段、格式检查）
5. 创建按钮状态管理（禁用/启用）

**验收标准**：
- ✓ 输入 ID 时，实时检查是否已存在
- ✓ 显示 ✓ 可用 / ✗ 已存在
- ✓ ID 已存在时，创建按钮禁用
- ✓ 预览区域实时显示 Agent 卡片效果

### Phase 3：角色选择和模板（1小时）

**目标**：实现角色卡片选择和模板

**任务**：
1. 实现角色卡片选择（6个预设角色）
2. 显示角色描述和特点
3. 点击卡片高亮选中
4. 加载角色模板数据

**验收标准**：
- ✓ 显示 6 个角色卡片（全栈、前端、后端、DevOps、测试、架构师）
- ✓ 点击卡片选中，高亮显示
- ✓ 预览区域显示选中的角色

### Phase 4：优化和完善（1-2小时）

**目标**：优化用户体验和错误处理

**任务**：
1. Emoji 选择器（可选，可以先用输入框）
2. 国际化支持（中英文）
3. 错误处理和用户反馈（Toast 提示）
4. 加载状态和动画（创建中...）
5. 键盘快捷键（Enter 提交，Esc 取消）

**验收标准**：
- ✓ 创建过程中显示加载状态
- ✓ 创建成功显示成功提示
- ✓ 创建失败显示错误信息
- ✓ 支持键盘快捷键

## 六、技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| 前端框架 | 原生 HTML/CSS/JS | 保持与现有 Webview 一致，不引入额外依赖 |
| 样式 | 复用 styles.css | 保持视觉一致性 |
| 消息通信 | VSCode Webview API | 官方推荐方式 |
| 验证 | 前端 + 后端双重验证 | 提升用户体验 + 保证数据安全 |
| Emoji 选择器 | 先用输入框，后续可升级 | 快速实现，后续优化 |

## 七、预期效果

### 用户体验提升

| 对比项 | 当前方式（InputBox） | Webview 方式 |
|--------|---------------------|--------------|
| 输入方式 | 一步步输入 | 一次性填写所有字段 |
| 预览 | 无 | 实时预览 Agent 卡片 |
| 验证 | 提交后才知道错误 | 实时验证，即时反馈 |
| 角色选择 | 下拉列表 | 卡片选择，更直观 |
| 取消操作 | 需要多次 Esc | 一次点击取消 |
| 视觉效果 | 原生对话框 | 自定义 UI，更美观 |

### 开发成本

- **总时间**：5-8 小时
- **Phase 1**：2-3 小时（基础功能）
- **Phase 2**：1-2 小时（验证和预览）
- **Phase 3**：1 小时（角色选择）
- **Phase 4**：1-2 小时（优化完善）

### 维护成本

- ✅ 代码结构清晰，易于维护
- ✅ 消息协议明确，易于扩展
- ✅ 独立模块，不影响现有功能

## 九、风险和注意事项

1. **兼容性**：确保 Webview 在不同 VSCode 版本中正常工作
2. **性能**：实时验证需要防抖，避免频繁请求
3. **安全性**：输入验证（前端 + 后端），防止注入攻击
4. **错误处理**：网络错误、创建失败等异常情况
5. **国际化**：支持中英文（后续可扩展更多语言）

## 十、总结

**核心优势**：
- ✅ 用户体验大幅提升（一次性填写 vs 多步输入）
- ✅ 实时验证和预览，减少错误
- ✅ 视觉效果更好，更符合现代 UI 标准
- ✅ 易于扩展（未来可做成完整的 Agent 管理面板）

**实现成本**：
- ⏱️ 5-8 小时开发时间
- 📦 约 500-800 行代码（HTML + JS + CSS）
- 🔧 可分阶段实现，逐步优化

**建议**：
1. 先实现 Phase 1（基础功能），验证可行性
2. 用户测试后，根据反馈优化
3. 逐步实现 Phase 2-4，完善功能
