# Agent 选择功能 - 需求设计与 UI 方案

## 需求背景

在 VSCode 扩展中增加 Agent 选择功能，允许用户：
1. 在聊天界面选择不同的 Agent
2. 创建新的 Agent，定义人设（开发相关角色）
3. Agent 选择与 VSCode Workspace 关联
4. 同一个 Workspace 下次打开时，默认使用上次选定的 Agent
5. 可以在 VSCode 设置中配置默认 Agent

## 技术架构

### SessionKey 格式
- **格式**：`agent:<agentId>:<windowId>`
- **示例**：`agent:dev2:panel-1`
- **说明**：切换 Agent 就是改变 sessionKey

### Agent 管理 API
- **优先**：Gateway RPC API
  - `agents.list` - 获取 Agent 列表
  - `agents.create` - 创建新 Agent
  - `agents.update` - 更新 Agent
  - `agents.delete` - 删除 Agent
  - `agents.files.get/set` - 读写 Agent 文件
- **备选**：CLI 命令（`openclaw agents list --json`, `openclaw agents add <id>`）
- **托底**：直接读取文件系统 `~/.openclaw/workspace/agents/`

### Workspace 关联
- **存储位置**：`~/.openclaw/workspace-mappings.json`
- **格式**：`{ "/path/to/workspace": "agentId" }`
- **优先级**：
  1. 项目目录下的 `.openclaw/agent` 文件（未实现）
  2. 全局映射文件
  3. VSCode 设置中的 `openclaw.defaultAgent`
  4. 固定默认值 `main`

### 工作区隔离
- **Mac/Unix**：`ln -s /path/to/project ~/.openclaw/workspace/agents/<agentId>/projects/<name>`
- **Windows**：
  - 优先：`mklink /J` (Junction，不需要管理员权限)
  - 备选：`mklink /D` (符号链接，需要管理员或开发者模式)
  - 失败提示：建议启用开发者模式

## UI 设计方案

### 1. Agent 选择器（主界面）

**位置**：聊天工具栏，模型选择器旁边

**UI 布局**：
```
┌─────────────────────────────────────────────────┐
│  OpenClaw Chat                            [×]   │
├─────────────────────────────────────────────────┤
│  [🐂 大牛 ▼]  [Claude Opus 4 ▼]  [⚙️]  [🔄]   │  ← 工具栏
│                                                 │
│  消息区域...                                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

**Agent 选择器按钮**：
- 显示：`[emoji] [name]` + 下拉箭头
- 示例：`🐂 大牛 ▼`
- 点击打开下拉菜单

**下拉菜单**：
```
┌─────────────────────────────┐
│ ✓ 🐂 大牛 (dev2)            │  ← 当前选中（有勾）
│   🦞 招财 (main)            │
│   ⚡️ 三万 (daily)           │
│   👷 Builder (builder)      │
│   📝 Writer (writer)        │
│   🧪 Tester (tester)        │
├─────────────────────────────┤
│ ➕ 创建新 Agent...          │
│ ⚙️ 管理 Agent...            │  ← 可选功能
└─────────────────────────────┘
```

**交互**：
- 点击某个 Agent → 切换到该 Agent
- 点击"创建新 Agent" → 打开创建对话框
- 点击"管理 Agent" → 打开管理界面（Phase 3）

## 技术细节

### CSS 类名
- `.agent-dropdown` - Agent 选择器容器
- `.agent-emoji` - Agent emoji 图标
- `.agent-option` - Agent 选项
- `.agent-option-check` - 选中标记
- `.agent-option-emoji` - 选项 emoji
- `.agent-option-info` - 选项信息
- `.agent-option-name` - 选项名称
- `.agent-option-id` - 选项 ID
- `.dropdown-separator` - 分隔线
- `.dropdown-action` - 操作按钮

### JavaScript 变量
- `currentAgentId` - 当前 Agent ID
- `agentsList` - Agent 列表
- `agentTrigger` - Agent 选择器按钮
- `agentLabel` - Agent 名称标签
- `agentEmoji` - Agent emoji 元素
- `agentOptionsEl` - Agent 选项容器
- `createAgentAction` - 创建 Agent 按钮

### 消息类型
- `getAgents` - 请求 Agent 列表
- `agentsList` - 返回 Agent 列表
- `switchAgent` - 切换 Agent
- `agentSwitched` - Agent 切换成功
- `createAgent` - 创建新 Agent
