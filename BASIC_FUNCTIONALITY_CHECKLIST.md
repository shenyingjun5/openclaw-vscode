## 可能的问题

### 1. 项目记忆自动发送消息（已修复）
**问题**：在 `_loadProjectMemory()` 中自动发送消息给 Agent，干扰对话流程
**修复**：已移除自动发送消息的逻辑（v0.2.19）
**验证**：
- [ ] 打开 VSCode，加载扩展
- [ ] 打开聊天面板
- [ ] 检查是否有自动发送的消息
- [ ] 预期：只显示"📚 已加载项目记忆"提示，不发送消息

### 2. Gateway 连接问题
**可能原因**：
- Gateway 未启动
- Gateway 版本不兼容
- WebSocket 连接失败

**验证步骤**：
- [ ] 检查 Gateway 是否运行：`openclaw gateway status`
- [ ] 检查 Gateway 版本：`openclaw --version`
- [ ] 在 VSCode 中打开聊天面板
- [ ] 查看连接状态指示器（应该显示"已连接"）
- [ ] 查看 VSCode 开发者工具控制台（Cmd+Shift+P -> "Developer: Toggle Developer Tools"）
- [ ] 检查是否有连接错误

### 3. 消息发送问题
**可能原因**：
- `sendChat` 方法参数错误
- `idempotencyKey` 生成问题
- RPC 调用失败

**验证步骤**：
- [ ] 在聊天面板中发送一条简单消息："你好"
- [ ] 检查是否显示"发送中..."状态
- [ ] 检查是否收到回复
- [ ] 查看控制台是否有错误

### 4. 会话管理问题
**可能原因**：
- SessionKey 格式错误
- 会话初始化失败
- 历史记录加载失败

**验证步骤**：
- [ ] 发送消息后，检查历史记录是否正确显示
- [ ] 刷新页面，检查历史记录是否保留
- [ ] 切换 Agent，检查是否正确加载新 Agent 的历史

### 5. Agent 切换问题
**可能原因**：
- Agent 列表获取失败
- SessionKey 更新失败
- 历史记录未清空

**验证步骤**：
- [ ] 点击 Agent 选择器
- [ ] 检查是否显示 Agent 列表
- [ ] 切换到另一个 Agent
- [ ] 检查是否显示切换成功提示
- [ ] 检查历史记录是否清空
- [ ] 发送消息，检查是否正常

## 测试步骤（按优先级）

### 第一步：验证 Gateway 连接
```bash
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 如果未运行，启动 Gateway
openclaw gateway start

# 3. 检查版本
openclaw --version
```

### 第二步：安装最新版本扩展
```bash
# 安装 v0.2.19（已修复项目记忆问题）
code --install-extension ~/Desktop/openclaw-vscode/openclaw-0.2.19.vsix
```

### 第三步：基础功能测试
1. 打开 VSCode
2. 打开一个项目
3. 打开 OpenClaw 聊天面板（点击侧边栏图标或 Cmd+Shift+P -> "OpenClaw: Open Chat Panel"）
4. 检查连接状态（右上角状态指示器）
5. 发送一条简单消息："你好"
6. 等待回复
7. 检查回复是否正常显示

### 第四步：Agent 切换测试
1. 点击 Agent 选择器（左下角）
2. 选择另一个 Agent
3. 检查是否切换成功
4. 发送消息测试

### 第五步：检查控制台日志
1. 打开 VSCode 开发者工具：Cmd+Shift+P -> "Developer: Toggle Developer Tools"
2. 切换到 Console 标签
3. 查看是否有错误或警告
4. 特别关注：
   - Gateway 连接错误
   - RPC 调用错误
   - 消息发送错误

## 常见问题和解决方案

### 问题 1：连接失败
**症状**：状态指示器显示"未连接"或"连接失败"
**解决方案**：
1. 检查 Gateway 是否运行：`openclaw gateway status`
2. 重启 Gateway：`openclaw gateway restart`
3. 检查配置：VSCode 设置 -> OpenClaw -> Gateway URL

### 问题 2：消息发送后无回复
**症状**：消息发送成功，但一直等待回复
**解决方案**：
1. 检查控制台是否有错误
2. 检查 Gateway 日志：`openclaw gateway logs`
3. 尝试重新连接：点击刷新按钮

### 问题 3：历史记录不显示
**症状**：发送消息后，历史记录中看不到
**解决方案**：
1. 点击刷新按钮
2. 检查 SessionKey 是否正确
3. 检查控制台错误

### 问题 4：Agent 切换失败
**症状**：点击 Agent 选择器无反应，或切换后仍是旧 Agent
**解决方案**：
1. 检查 Agent 是否存在：`openclaw agents list`
2. 检查控制台错误
3. 重新加载 VSCode 窗口
